# Light Research Test Protocol

## Purpose

This protocol defines evidence requirements for future lighting-related MMI research.

It is a shell only. It does not contain confirmed values or operational instructions.

## Protocol stages

| Stage | Goal | Vehicle change? | Status |
|---|---|---:|---|
| L0 | Collect module context | No | 🟠 TO VERIFY |
| L1 | Capture MMI screen evidence | No | 🟠 TO VERIFY |
| L2 | Capture DTC baseline | No | 🟠 TO VERIFY |
| L3 | Compare menu visibility with module context | No | 🟠 TO VERIFY |
| L4 | Controlled technical test | Not defined here | 🔴 BLOCKED |

## Required evidence before any future technical test

| Evidence | Required | Status |
|---|---:|---|
| Full Auto-Scan | Yes | 🟠 TO CAPTURE |
| BCM/J519 identity | Yes | 🟠 TO CAPTURE |
| Headlight range identity if present | Yes for related claims | 🟠 TO CAPTURE |
| Gateway installation list | Yes | 🟠 TO CAPTURE |
| Original backup data | Yes | 🟠 TO CAPTURE |
| DTC before/after | Yes | 🟠 TO CAPTURE |
| MMI screen evidence | Yes | 🟠 TO CAPTURE |
| rollback evidence | Yes for any future change | 🟠 TO CAPTURE |

## Review questions

| Question | Required answer |
|---|---|
| Which menu item is observed? | Exact menu path |
| Which modules may be related? | Address and module name |
| What evidence is available? | File/photo/log link |
| What remains unknown? | Explicit unknown list |
| Is any result variant-specific? | Must be stated |

## Current decision

Only read-only evidence planning is allowed in this protocol.
